<h2><span class="mw-headline" id="Installing_devkitPro_Pacman">Installing devkitPro Pacman</span></h2>
<p>devkitPro provided tools and libraries are managed by the rather wonderful <a rel="nofollow" class="external text" href="https://wiki.archlinux.org/index.php/pacman">Arch Linux pacman</a>. We provide <a rel="nofollow" class="external text" href="https://github.com/devkitPro/pacman/releases/latest">our own binaries</a> as .pkg for OSX and via our own apt repository for debian based linux distributions.
</p>
<h2><span class="mw-headline" id="Customising_Existing_Pacman_Install">Customising Existing Pacman Install</span></h2>
<p><span style="color: red;"><b>Note:</b> These instructions are for systems which come with pacman already installed i.e. Arch or Msys2. <b>Do Not</b> follow these instructions if you have dkp-pacman or used the devkitPro installer</span>
</p><p>For users already using a distro which provides pacman please follow these instructions. This also applies to Alpine, Fedora, Gentoo, and Void below as well as users who wish to use an existing msys2 install. If you're using a pre-existing msys2 install run the commands without sudo. Please also set environment variables as follows&#160;:- 
</p>
<pre>   DEVKITPRO=/opt/devkitpro
   DEVKITARM=/opt/devkitpro/devkitARM
   DEVKITPPC=/opt/devkitpro/devkitPPC
</pre>
<p>Then import the key which is used to validate the packages. There seem to be issues with keyservers so we specify one here but you may need to look for a working one.
</p>
<pre>  sudo pacman-key --recv BC26F752D25B92CE272E0F44F7FD5492264BB9D0 --keyserver keyserver.ubuntu.com
  sudo pacman-key --lsign BC26F752D25B92CE272E0F44F7FD5492264BB9D0
</pre>
<p>Now install the devkitpro keyring. Msys2 users should ignore the sudo part - i.e. type the command line without sudo.
</p>
<pre>   sudo pacman -U <a rel="nofollow" class="external free" href="https://pkg.devkitpro.org/devkitpro-keyring.pkg.tar.zst">https://pkg.devkitpro.org/devkitpro-keyring.pkg.tar.zst</a>
</pre>
<p>On some systems there's an issue running the post install script resulting in an error message like `call to execv failed (No such file or directory)` so then if this happens then run
</p>
<pre>   sudo pacman-key --populate devkitpro
</pre>
<p>Add the devkitPro repositories. <span style="color: red;"><b>Please note that you will require two - one for the libraries and another for the host specific tools.</b></span>
</p><p>edit /etc/pacman.conf &amp; add these lines
</p>
<pre>[dkp-libs]
Server = <a rel="nofollow" class="external free" href="https://pkg.devkitpro.org/packages">https://pkg.devkitpro.org/packages</a>
</pre>
<p>for glibc base linux systems also add 
</p>
<pre>[dkp-linux]
Server = <a rel="nofollow" class="external free" href="https://pkg.devkitpro.org/packages/linux/$arch/">https://pkg.devkitpro.org/packages/linux/$arch/</a>
</pre>
<p>for musl based linux systems like Alpine and Void use this instead of dkp-linux 
</p>
<pre>[dkp-linux-musl]
Server =  <a rel="nofollow" class="external free" href="https://pkg.devkitpro.org/packages/linux-musl/$arch/">https://pkg.devkitpro.org/packages/linux-musl/$arch/</a>
</pre>
<p>for existing msys2 systems use this one instead of dkp-linux above.
</p>
<pre>[dkp-windows]
Server = <a rel="nofollow" class="external free" href="https://pkg.devkitpro.org/packages/windows/$arch/">https://pkg.devkitpro.org/packages/windows/$arch/</a>
</pre>
<p>Now resync the database and update installed packages.
</p>
<pre>    sudo pacman -Syu
</pre>
<h3><span class="mw-headline" id="Apine">Apine</span></h3>
<p>On Alpine you can obtain pacman using the command <b>sudo apk install pacman</b> and edit the pacman configuration files as you would for Arch based linux distros and pre-existing msys2 installs. Run <b>pacman-key --init</b> then follow the <a rel="nofollow" class="external text" href="https://devkitpro.org/wiki/devkitPro_pacman#Customising_Existing_Pacman_Install">customising existing pacman instructions</a> above.
</p>
<h3><span class="mw-headline" id="Debian_and_derivatives">Debian and derivatives</span></h3>
<p>On Debian based systems we have a script that sets up the repository for devkitPro pacman. You can simply download and run this but do please read the code for your own peace of mind.
</p>
<pre>   wget -U "dkp-apt" <a rel="nofollow" class="external free" href="https://apt.devkitpro.org/install-devkitpro-pacman">https://apt.devkitpro.org/install-devkitpro-pacman</a>
   chmod +x ./install-devkitpro-pacman
   sudo ./install-devkitpro-pacman
</pre>
<p>The apt repository can also be used on WSL but you may need to make a symlink for /etc/mtab. See <a rel="nofollow" class="external free" href="https://github.com/Microsoft/WSL/issues/150">https://github.com/Microsoft/WSL/issues/150</a>
</p>
<pre>   sudo ln -s /proc/self/mounts /etc/mtab
</pre>
<p>Once you've done that then check out <a rel="nofollow" class="external text" href="https://devkitpro.org/wiki/devkitPro_pacman#Using_Pacman">Using Pacman</a> below.
</p>
<h3><span class="mw-headline" id="Fedora">Fedora</span></h3>
<p>On Fedora you can obtain pacman using the command <b>sudo dnf install pacman</b> and edit the pacman configuration files as you would for Arch based linux distros , and pre-existing msys2 installs. Run <b>pacman-key --init</b> then follow the <a rel="nofollow" class="external text" href="https://devkitpro.org/wiki/devkitPro_pacman#Customising_Existing_Pacman_Install">customising existing pacman instructions</a> above.
</p>
<h3><span class="mw-headline" id="Gentoo">Gentoo</span></h3>
<p>Enable the gentoo-zh overlay:
</p>
<pre>   eselect repository enable gentoo-zh
</pre>
<p>Update repositories:
</p>
<pre>   emerge --sync
</pre>
<p>Install pacman: emerge --ask sys-apps/pacman
</p><p>Note. The ~amd64 flag might need to be added to accept_keywords for sys-apps/pacman, app-crypt/archlinux-keyring and sys-apps/pacman-mirrorlist
</p>
<pre>   echo "sys-apps/pacman ~amd64" &gt; /etc/portage/package.accept_keywords/pacman
   echo "app-crypt/archlinux-keyring ~amd64" &gt;&gt; /etc/portage/package.accept_keywords/pacman
   echo "sys-apps/pacman-mirrorlist ~amd64" &gt;&gt; /etc/portage/package.accept_keywords/pacman
</pre>
<p>Optional (Recommended) Fetch the pacman mirrorlist:
</p>
<pre>   emerge --ask sys-apps/pacman-mirrorlist
</pre>
<p>Select mirrors or create the list from scratch:
</p>
<pre>   nano /etc/pacman.d/mirrorlist
</pre>
<p>Set up the keyring:
</p>
<pre>   pacman-key --init
</pre>
<p>In /etc/pacman.conf:
</p>
<pre>   nano /etc/pacman.conf
</pre>
<p>Set RootDir to root (/). The gentoo-zh pacman package recommends against this but it does not create any problems in our case as it will only be used to 
install dkP related packages. Comment out an XferCommand (either curl or wget). Proceed with the <a rel="nofollow" class="external text" href="https://devkitpro.org/wiki/devkitPro_pacman#Customising_Existing_Pacman_Install">customising existing pacman instructions</a> below.
</p>
<h3><span class="mw-headline" id="macOS">macOS</span></h3>
<p>On macOS run the .pkg installer from Terminal with
</p>
<pre>   sudo installer -pkg devkitpro-pacman-installer.pkg -target /
</pre>
<p>Then install the Xcode command line tools if you haven't already
</p>
<pre>   xcode-select --install
</pre>
<p>Reboot your mac to have ennviromnent variables set then then head to <a rel="nofollow" class="external text" href="https://devkitpro.org/wiki/devkitPro_pacman#Using_Pacman">Using Pacman</a> below.
</p>
<h3><span class="mw-headline" id="openSUSE">openSUSE</span></h3>
<p>On openSUSE you can obtain pacman through <a rel="nofollow" class="external text" href="https://software.opensuse.org/package/pacman">Experimental or Community packages</a> hosted on openSUSE Build Service. Pick the latest available version and install manually or by using <a rel="nofollow" class="external text" href="https://en.opensuse.org/openSUSE:One_Click_Install">1 Click Install</a> method. After installation you need to add alpm user by executing
</p>
<pre>   sudo useradd --system --no-create-home --home-dir / --shell /usr/sbin/nologin alpm
</pre>
<p>If successful, you'll see alpm user added in your passwd file
</p>
<pre>   # alpm:x:470:470::/:/usr/sbin/nologin
   cat /etc/passwd
</pre>
<p>Uncomment preferable server (e.g. first entry in Worldwide section) in <b>/etc/pacman.d/mirrorlist</b>
</p>
<pre>   ## Worldwide
   Server = <a rel="nofollow" class="external free" href="https://geo.mirror.pkgbuild.com/$repo/os/$arch">https://geo.mirror.pkgbuild.com/$repo/os/$arch</a>
</pre>
<p>Run <b>pacman-key --init</b> then follow the <a rel="nofollow" class="external text" href="https://devkitpro.org/wiki/devkitPro_pacman#Customising_Existing_Pacman_Install">customising existing pacman instructions</a> above.
</p>
<h3><span class="mw-headline" id="Void_Linux">Void Linux</span></h3>
<p>On Void you can obtain pacman using the command <b>sudo xbps-install -S pacman</b> and edit the pacman configuration files as you would for Arch based linux distros and pre-existing msys2 installs. Run <b>pacman-key --init</b> then follow the <a rel="nofollow" class="external text" href="https://devkitpro.org/wiki/devkitPro_pacman#Customising_Existing_Pacman_Install">customising existing pacman instructions</a> above.
</p>
<h3><span class="mw-headline" id="Windows">Windows</span></h3>
<p>We provide an <a rel="nofollow" class="external text" href="https://github.com/devkitPro/installer/releases/latest">installer for 64bit windows</a> which sets up a customised msys2 install already set up with the devkitPro packages.Unfortunately msys2 are no longer supporting 32bit and we advise moving to 64 bit if at all possible. You can still obtain the <a rel="nofollow" class="external text" href="http://repo.msys2.org/distrib/msys2-i686-latest.exe">32bit msys2 installer</a> and follow the manual instructions below.
</p>
<h2><span class="mw-headline" id="Using_Pacman">Using Pacman</span></h2>
<p>The customised pacman we ship for OSX and debian based distros is entirely self-contained within /opt/devkitpro/pacman. To avoid polluting the system and avoid clashes with system tools or games which may share the same name as pacman binaries we install helper scripts in /usr/local/bin which temporarily set path and forward to the custom binaries. <span style="color: red;"><b>This means you substitute dkp-pacman for pacman on those systems. Do not add /opt/devkitpro/pacman/bin to your system path, this will cause issues.</b><span>
</span></span></p>
<h3><span class="mw-headline" id="Updating_Databases">Updating Databases</span></h3>
<p>To resync the databases before obtaining updates, this command is similar to <b>apt-get update</b>
</p>
<pre>   sudo (dkp-)pacman -Sy
</pre>
<p>To update installed packages, this command is similar to <b>apt-get upgrade</b>
</p>
<pre>   sudo (dkp-)pacman -Syu
</pre>
<h3><span class="mw-headline" id="Installing_packages">Installing packages</span></h3>
<p>You can list all available packages with
</p>
<pre>   (dkp)-pacman -Sl
</pre>
<p>To show only the libraries supplied by devkitPro append dkp-libs
</p>
<pre>   (dkp-)pacman -Sl dkp-libs
</pre>
<p>You can search for packages using
</p>
<pre>   (dkp-)pacman -Ss &lt;search string&gt;
</pre>
<h3><span class="mw-headline" id="Predefined_Groups">Predefined Groups</span></h3>
<p>We provide several convenient groups which may be installed using <b>sudo (dkp-)pacman -S &lt;group name&gt;</b>
</p>
<pre>- gp32-dev
- gp2x-dev
- gba-dev 
- nds-dev
- 3ds-dev
- gamecube-dev
- wii-dev
- wiiu-dev
- switch-dev
</pre>
<p>To list packages use `dkp-pacman -Sl`
To install a package use `sudo dkp-pacman -S &lt;name of package&gt;`
`sudo dkp-pacman -R &lt;name of package&gt;` will remove an already installed package.
</p>
